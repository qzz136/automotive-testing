代码示例

ECU仿真测试场景

from tsmaster import (
    ECUSimulationScenario,
    TestStep,
    MessageFrame,
    StepType,
)

scenario = ECUSimulationScenario(
    scenario_name="Digital Key Welcome Test",
    channel=0,
    steps=[
        TestStep(step_id="init", step_type=StepType.INIT, order=0),
        TestStep(
            step_id="zone0",
            step_type=StepType.SMART_CAR_ZONE,
            order=1,
            zone_value=0
        ),
        TestStep(
            step_id="cyclic_500",
            step_type=StepType.START_CYCLIC,
            order=2,
            period_ms=700,
            message=MessageFrame(channel=0, identifier="0x500", data=[0]*8)
        ),
        TestStep(
            step_id="check_signals",
            step_type=StepType.CHECK_SIGNALS,
            order=3,
            check_dbc_path="D31L_15.3_CAN4_DKC_20251204_Draft.dbc",
            check_message_ids=["0x251"],
            conditions=[
                {"signal": "P_DKey_Welcome", "operator": "==", "value": 1},
            ]
        ),
    ]
)

CAN信号编码

from tsmaster import encode_can_signal
from tsmaster.models import SignalInput

dbc_path = "D31L_15.3_CAN4_DKC_20251204_Draft.dbc"
signals = [
    SignalInput(signal="PwrSta", value=3),
    SignalInput(signal="Voltage", value=12.5),
]

result = encode_can_signal(dbc_path, signals)
# Returns: {"status": "success", "frame_id": "0x116", "data": [...]}

使用场景

智能小车测试

通过TCP协议控制智能小车移动，实现自动化区域测试。支持单次按键、持续开关、区域移动等多种控制模式。

  • zone_value: 区域移动目标
  • switch_value + keytime_ms: 按键控制
  • enable_disable: 持续控制开关
TCP
智能硬件

信号时序检查

验证信号在指定时间或多帧内保持特定值，确保ECU响应符合时序要求。支持回溯分析和条件组合检查。

  • hold_max_frames: 连续帧数要求
  • hold_duration_ms: 持续时间要求
  • check_lookback_ms: 回溯时间窗口
DBC
