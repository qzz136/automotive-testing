概述

什么是车载DKC测试？

DKC（数字钥匙控制器）是汽车数字钥匙系统的核心，负责无钥匙进入、启动授权、身份认证等功能。 通过OpenCode Agent，测试人员可以用自然语言描述测试场景，Agent自动执行复杂的测试序列。

📡

CAN/CANFD 报文收发

支持单帧发送、周期发送、报文接收等多种通信模式，覆盖车载网络测试的核心需求。

🔍

DBC信号解码

使用DBC文件定义报文格式，自动将原始字节解码为有意义的信号值，支持复杂的数据结构。

⏱️

时序信号检查

验证信号在指定时间窗口内保持特定值，支持多信号联合检查，确保时序逻辑正确。

🤖

智能设备控制

集成智能小车和机械臂控制，实现物理世界的自动化测试操作。

📝

自动日志记录

测试过程中自动记录BLF/ASC格式的报文日志，便于事后分析和问题追溯。

🎯

场景化测试

通过定义测试场景和步骤序列，轻松构建复杂的测试用例，支持步骤间的数据传递。

@mcp.tool(
    name="tsmaster_run_simulation",
    annotations={
        "title": "执行DKC仿真测试场景",
    },
)
async def tsmaster_run_simulation(scenario: ECUSimulationScenario) -> str:
    """执行测试序列"""
    _ensure_connected()
    sorted_steps = sorted(scenario.steps, key=lambda s: s.order)
    step_results = []
    
    for step in sorted_steps:
        result = _execute_step(step, scenario.channel)
        step_results.append(result)
    
    return json.dumps({
        "scenario_name": scenario.scenario_name,
        "status": "completed",
        "step_results": step_results
    })

# tsmaster/models.py - 数据模型
from pydantic import BaseModel

class StepType(Enum):
    INIT = "init"
    SEND_SINGLE = "send_single"
    START_CYCLIC = "start_cyclic"
    WAIT = "wait"
    RECEIVE = "receive"
    SMART_CAR_ZONE = "smart_car_zone"
    MACHINE_ARM_ROTATION = "machine_arm_rotation"
    NFC_START = "nfc_start"
    CHECK_SIGNALS = "check_signals"

class MessageFrame(BaseModel):
    channel: int = 0
    identifier: Union[int, str]
    data: List[int]
    is_extended_id: bool = False

class TestStep(BaseModel):
    step_id: str
    step_type: StepType
    order: int
    message: Optional[MessageFrame] = None
    duration_ms: Optional[int] = None
    zone_value: Optional[int] = None
    angle: Optional[int] = None
    conditions: Optional[List[Dict]] = None

class ECUSimulationScenario(BaseModel):
    scenario_name: str
    channel: int = 0
    steps: List[TestStep]
# tsmaster/executor.py - 测试执行器
def _execute_step(step: TestStep, channel: int) -> StepResult:
    """执行单个测试步骤"""
    if step.step_type == StepType.INIT:
        # 初始化，启动BLF记录
        return _handle_init(step)
    
    elif step.step_type == StepType.SEND_SINGLE:
        # 发送单帧报文
        return _handle_send_single(step, channel)
    
    elif step.step_type == StepType.SMART_CAR_ZONE:
        # 控制智能小车
        success = send_zone_value(step.zone_value)
        return StepResult(status="passed" if success else "failed")
    
    elif step.step_type == StepType.CHECK_SIGNALS:
        # 检查信号条件
        return _handle_check_signals(step, channel)
核心功能

测试步骤类型

系统支持12+种测试步骤类型，覆盖从初始化到信号验证的完整测试流程。

🔌
INIT
初始化

初始化测试环境，启动TSMaster报文记录（BLF格式），设置日志文件路径。

📤
SEND_SINGLE
报文发送

发送单帧CAN/CANFD报文，支持标准帧和扩展帧，可配置数据长度码。

🔄
START_CYCLIC
周期发送

启动周期报文发送，可配置发送周期（10ms-60s），模拟持续信号输出。

📥
RECEIVE
报文接收

接收并验证总线报文，可指定期望接收的报文ID列表，支持超时检测。

⏸️
WAIT
延时等待

等待指定时长，用于模拟真实场景中的时间延迟，或等待系统响应。

🚗
SMART_CAR_ZONE
智能小车

控制智能小车移动到指定区域，用于物理世界的自动化测试操作。

🦾
MACHINE_ARM_ROTATION
机械臂

控制机械臂旋转到指定角度（0-180度），支持NFC刷卡等物理操作。

🔍
CHECK_SIGNALS
信号检查

检查信号条件是否满足，支持时序检查（连续N帧保持值）和容差比较。

            order=1,

                channel=0,
                identifier="0x500",

                is_extended_id=False

        ),
        
        # 步骤3: 智能小车移动到区域2 (模拟钥匙靠近)

            step_id="car_move_zone2",
            step_type=StepType.SMART_CAR_ZONE,
            order=2,
            zone_value=2

            step_id="wait_auth",
